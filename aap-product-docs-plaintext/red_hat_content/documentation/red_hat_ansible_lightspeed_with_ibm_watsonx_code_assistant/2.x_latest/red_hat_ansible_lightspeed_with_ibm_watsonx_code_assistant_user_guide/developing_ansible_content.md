# 2. Getting started with Red Hat Ansible Lightspeed
## 2.2. Quick start for developers
### 2.2.2. Developing Ansible content




After installing and configuring the Ansible VS Code extension, developers can create custom Ansible content.

You can perform the following tasks:

-  [Start a free 90-day trial of Red Hat Ansible Lightspeed](https://docs.redhat.com/en/documentation/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant/2.x_latest/html-single/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant_user_guide/index#start-lightspeed-trial_start-lightspeed-trial)

This is especially useful as you **do not need** a trial or paid subscription to IBM watsonx Code Assistant. After your 90-day trial expires, you must set up the cloud service or on-premise deployment to continue accessing the Ansible Lightspeed service.


-  [Create single-task or multitask code recommendations](https://docs.redhat.com/en/documentation/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant/2.x_latest/html-single/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant_user_guide/index#con-task-recommendations_developing-ansible-content)

You can generate single-task or multitask code recommendations for your task intent using the Ansible VS Code extension.


-  [Create Ansible playbooks and view playbook explanations](https://docs.redhat.com/en/documentation/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant/2.x_latest/html-single/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant_user_guide/index#playbook-generation_developing-ansible-content)

You can create Ansible playbooks and view explanations for new or existing playbooks by using the Ansible VS Code extension.


-  [Create roles within a collection and view role explanations](https://docs.redhat.com/en/documentation/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant/2.x_latest/html-single/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant_user_guide/index#role-creation_developing-ansible-content)

You can generate roles within collections using the Ansible VS Code extension and view explanations for existing roles.
