### 5.1.3. Regions and availability zones




Each supported region is paired with a companion AWS region where backup data is stored in the event of a primary region catastrophe that requires restoration in another AWS region.

Refer to [Backup and disaster recovery](https://docs.redhat.com/en/documentation/ansible_on_clouds/2.x/html-single/red_hat_ansible_automation_platform_service_on_aws/index#con-saas-backup-and-disaster-recovery) for the list of supported and backup regions.
