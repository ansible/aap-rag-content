# 1. Key functionality and concepts
## 1.10. Using this guide




After you have installed Ansible Automation Platform 2.5 and have become familiar with the dashboard, use this document to explore further options for setup and daily use. This guide is structured so that you can select the path that is most appropriate to you and your role within your organization. We also encourage you to explore the other paths outlined in this guide to learn how Ansible empowers users with various roles and objectives to build and customize automation tasks.

Select one of the following paths to continue getting started:

- If you are a systems administrator configuring authentication and setting up teams and organizations, see [Getting started as a platform administrator](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.5/html-single/getting_started_with_ansible_automation_platform/index#assembly-gs-platform-admin) .
- If you are a developer setting up development environments, creating playbooks, rulebooks, roles, or projects, see [Getting started as an automation developer](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.5/html-single/getting_started_with_ansible_automation_platform/index#assembly-gs-auto-dev) .
- If you are an operator using playbooks, publishing custom content, creating projects, and creating and using inventories, see [Getting started as an automation operator](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.5/html-single/getting_started_with_ansible_automation_platform/index#assembly-gs-auto-op) .
