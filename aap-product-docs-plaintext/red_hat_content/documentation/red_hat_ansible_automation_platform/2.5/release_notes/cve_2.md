# 10. Patch releases
## 10.2. Ansible Automation Platform patch release October 22, 2025
### 10.2.2. CVE




With this update, the following CVEs have been addressed:

[CVE-2025-59682](https://access.redhat.com/security/cve/cve-2025-59682)  `python-django` : Potential partial directory-traversal via `archive.extract()` .(AAP-54753)

[CVE-2025-59343](https://access.redhat.com/security/cve/cve-2025-59343)  `automation-gateway` : `tar-fs` symlink validation bypass.(AAP-54393)

[CVE-2025-7647](https://access.redhat.com/security/cve/cve-2025-7647)  `ansible-automation-platform-25/lightspeed-chatbot-rhel8` : insecure temporary file handling in `run-llama/llama_index` .(AAP-55890)

[CVE-2025-6985](https://access.redhat.com/security/cve/cve-2025-6985)  `ansible-automation-platform-25/lightspeed-rhel8` : XXE vulnerability in `langchain-text-splitters` .(AAP-55257)

[CVE-2025-59682](https://access.redhat.com/security/cve/cve-2025-59682)  `ansible-automation-platform-25/lightspeed-rhel8` : Potential partial directory-traversal via `archive.extract()` .(AAP-54720)

[CVE-2025-59681](https://access.redhat.com/security/cve/cve-2025-59681)  `ansible-automation-platform-25/lightspeed-rhel8` : Potential SQL injection in `QuerySet.annotate()` , `alias()` , `aggregate()` , and `extra()` on MySQL and MariaDB1.(AAP-54661)
