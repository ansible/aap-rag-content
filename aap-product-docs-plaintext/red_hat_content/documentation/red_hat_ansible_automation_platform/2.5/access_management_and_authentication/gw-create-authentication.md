# 2. Configuring authentication in the Ansible Automation Platform
## 2.3. Creating an authentication method




Creating an authenticator involves the following procedures:

-  [Selecting an authentication type](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.5/html-single/access_management_and_authentication/index#gw-select-auth-type) , where you select the type of authenticator plugin you want to configure, including the authentication details for the authentication type selected.
-  [Mapping](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.5/html-single/access_management_and_authentication/index#gw-define-rules-triggers) , where you define mapping rule types and triggers to control access to the system, and [Mapping order](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.5/html-single/access_management_and_authentication/index#gw-adjust-mapping-order) , where you can define the mapping precedence.

Note
Mapping order is only available if you have defined one or more authenticator maps.
