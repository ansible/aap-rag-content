# 4. Managing access with role-based access control
## 4.2. Teams
### 4.2.2. Creating a team




You can create new teams, assign an organization to the team, and manage the users and administrators assigned to each team. Users on a team inherit the permissions and roles assigned to the team. To assign a user or administrator to a team, the user must have already been created. See [Assigning users to a team](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.6/html/access_management_and_authentication/gw-managing-access#proc-gw-team-add-user) or [Assigning administrators to a team](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.6/html/access_management_and_authentication/gw-managing-access#proc-gw-add-admin-team) for more information.

**Procedure**

1. From the navigation panel, selectAccess Management→Teams.
1. ClickCreate team.
1. Enter a **Name** and optionally give a **Description** for the team.
1. Select an **Organization** to be associated with this team.

Note
Each team can only be assigned to one organization.




1. ClickCreate team. The **Details** page opens, where you can review and edit your team information and access.
