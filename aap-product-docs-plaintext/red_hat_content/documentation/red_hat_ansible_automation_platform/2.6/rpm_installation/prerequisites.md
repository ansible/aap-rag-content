# 5. Disconnected installation
## 5.1. Prerequisites




Before installing Ansible Automation Platform on a disconnected network, you must meet the following prerequisites:

- A subscription manifest that you can upload to the platform.


For more information, see [Obtaining a manifest file](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.6/html/access_management_and_authentication/assembly-gateway-licensing#assembly-aap-obtain-manifest-files) .

- The Ansible Automation Platform setup bundle at [Customer Portal](https://access.redhat.com/downloads/content/480/ver=2.6/rhel---9/2.6/x86_64/product-software) is downloaded.
- The [DNS records](https://docs.ansible.com/ansible/latest/collections/community/general/nsupdate_module.html) for the automation controller and private automation hub servers are created.
