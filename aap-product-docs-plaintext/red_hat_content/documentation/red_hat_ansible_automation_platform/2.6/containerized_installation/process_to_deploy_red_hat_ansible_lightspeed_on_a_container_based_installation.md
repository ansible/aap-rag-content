# 6. Deploying Red Hat Ansible Lightspeed on containerized Ansible Automation Platform
## 6.1. Overview
### 6.1.7. Process to deploy Red Hat Ansible Lightspeed on a container-based installation

| Task | Description |
| --- | --- |
| <br>  Deploy Red Hat Ansible Lightspeed during a container-based installation of Ansible Automation Platform | <br>  An Ansible Automation Platform administrator who wants to deploy Red Hat Ansible Lightspeed for all Ansible users in the organization. <br>  Perform the following tasks:    <br> [Configure the Red Hat Ansible Lightspeed variables in the inventory file](#proc-configure-lightspeed-variables_deploying-lightspeed-containerized-install "6.2.&nbsp;Configuring Red Hat Ansible Lightspeed variables in the inventory file"). [Install](#installing-containerized-aap "Chapter&nbsp;8.&nbsp;Installing containerized Ansible Automation Platform") or [upgrade](#updating-containerized-ansible-automation-platform "9.1.&nbsp;Upgrading containerized Ansible Automation Platform") to containerized Ansible Automation Platform 2.6.   If you want to install the Ansible Lightspeed coding assistant, [configure the Ansible VS Code extension](#proc-configure-vs-code-containerized-install_deploying-lightspeed-containerized-install "6.5.&nbsp;Configuring the Ansible VS Code extension").   Optional: [Change your LLM model](#proc-change-llm-model-containerized-install_deploying-lightspeed-containerized-install "6.3.&nbsp;Changing your LLM model") if you want to use a different LLM provider after deploying Red Hat Ansible Lightspeed. |
| <br>  Access and use the Ansible Lightspeed intelligent assistant | <br>  All Ansible users within the organization who want to [use the Ansible Lightspeed intelligent assistant](#con-using-chatbot_deploying-lightspeed-containerized-install "6.4.&nbsp;Using the Ansible Lightspeed intelligent assistant") to get answers to their questions about the Ansible Automation Platform. |
| <br>  Access and use the Ansible Lightspeed coding assistant | <br>  All Ansible users within the organization who want to use the coding assistant to [develop Ansible content](https://docs.redhat.com/en/documentation/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant/2.x_latest/html/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant_user_guide/developing-ansible-content_lightspeed-user-guide):    <br> [Single task or multitask recommendations](https://docs.redhat.com/en/documentation/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant/2.x_latest/html/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant_user_guide/developing-ansible-content_lightspeed-user-guide#con-task-recommendations_developing-ansible-content) [Create playbooks and view playbook explanations](https://docs.redhat.com/en/documentation/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant/2.x_latest/html/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant_user_guide/developing-ansible-content_lightspeed-user-guide#playbook-generation_developing-ansible-content) [Create roles and view role explanations](https://docs.redhat.com/en/documentation/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant/2.x_latest/html/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant_user_guide/developing-ansible-content_lightspeed-user-guide#role-creation_developing-ansible-content) |
