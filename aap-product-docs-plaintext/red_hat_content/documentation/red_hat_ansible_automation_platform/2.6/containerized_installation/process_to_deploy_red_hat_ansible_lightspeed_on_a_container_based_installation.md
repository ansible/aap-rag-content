# 6. Deploying Red Hat Ansible Lightspeed on containerized Ansible Automation Platform
## 6.1. Overview
### 6.1.7. Process to deploy Red Hat Ansible Lightspeed on a container-based installation




| Task | Description |
| --- | --- |
| Deploy Red Hat Ansible Lightspeed during a container-based installation of Ansible Automation Platform | An Ansible Automation Platform administrator who wants to deploy Red Hat Ansible Lightspeed for all Ansible users in the organization.

Perform the following tasks:

1.  [Configure the Red Hat Ansible Lightspeed variables in the inventory file](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.6/html-single/containerized_installation/index#proc-configure-lightspeed-variables_deploying-lightspeed-containerized-install) .
1.  [Install](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.6/html-single/containerized_installation/index#installing-containerized-aap) or [upgrade](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.6/html-single/containerized_installation/index#updating-containerized-ansible-automation-platform) to containerized Ansible Automation Platform 2.6.
1. If you want to install the Ansible Lightspeed coding assistant, [configure the Ansible VS Code extension](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.6/html-single/containerized_installation/index#proc-configure-vs-code-containerized-install_deploying-lightspeed-containerized-install) .
1. Optional: [Change your LLM model](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.6/html-single/containerized_installation/index#proc-change-llm-model-containerized-install_deploying-lightspeed-containerized-install) if you want to use a different LLM provider after deploying Red Hat Ansible Lightspeed. |
| Access and use the Ansible Lightspeed intelligent assistant | All Ansible users within the organization who want to [use the Ansible Lightspeed intelligent assistant](https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/2.6/html-single/containerized_installation/index#con-using-chatbot_deploying-lightspeed-containerized-install) to get answers to their questions about the Ansible Automation Platform. |
| Access and use the Ansible Lightspeed coding assistant | All Ansible users within the organization who want to use the coding assistant to [develop Ansible content](https://docs.redhat.com/en/documentation/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant/2.x_latest/html/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant_user_guide/developing-ansible-content_lightspeed-user-guide) :

-  [Single task or multitask recommendations](https://docs.redhat.com/en/documentation/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant/2.x_latest/html/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant_user_guide/developing-ansible-content_lightspeed-user-guide#con-task-recommendations_developing-ansible-content)
-  [Create playbooks and view playbook explanations](https://docs.redhat.com/en/documentation/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant/2.x_latest/html/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant_user_guide/developing-ansible-content_lightspeed-user-guide#playbook-generation_developing-ansible-content)
-  [Create roles and view role explanations](https://docs.redhat.com/en/documentation/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant/2.x_latest/html/red_hat_ansible_lightspeed_with_ibm_watsonx_code_assistant_user_guide/developing-ansible-content_lightspeed-user-guide#role-creation_developing-ansible-content) |
