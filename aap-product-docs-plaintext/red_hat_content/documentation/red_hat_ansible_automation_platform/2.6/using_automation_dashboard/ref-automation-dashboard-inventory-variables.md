# 2. Appendix
## 2.1. Inventory variables

The following variables control how automation dashboard interacts with remote hosts.

| Inventory variable | Description |
| --- | --- |
| <br> `aap_auth_provider_name` | <br>  Natural language name shown on the login page. Default: `Ansible Automation Platform` |
| <br> `aap_auth_provider_protocol` | <br>  Enter http or https |
| <br> `aap_auth_provider_aap_version` | <br>  Enter one value - the version of Ansible Automation Platform that automation dashboard will connect to. Valid values are 2.4, 2.5 or 2.6 |
| <br> `aap_auth_provider_host` | <br>  Ansible Automation Platform IP or DNS name, with optional port |
| <br> `aap_auth_provider_check_ssl` | <br>  Enforce TLS check or not |
| <br> `aap_auth_provider_client_id` | <br>  Ansible Automation Platform OAuth2 application `client_id`. Required for SSO authentication. |
| <br> `aap_auth_provider_client_secret` | <br>  Ansible Automation Platform OAuth2 application `client_secret` |
| <br> `initial_sync_days` | <br>  Requests x number of days of old data, counting from "today" |
| <br> `initial_sync_since` | <br>  Requests data from the specified date until "today" |
| <br> `dashboard_update_secret` | <br>  Forces regeneration of autogenerated Podman secrets. Store the password for database access by using a Podman secret. Set `dashboard_update_secret` to true if you changed the `dashboard_pg_password` in inventory. |
| <br> `nginx_disable_https` | <br>  Enables use of http instead of https for automation dashboard |
| <br> `nginx_http_port` | <br>  Configures the HTTP port for automation dashboard |
| <br> `nginx_https_port` | <br>  Configures the HTTPS port for automation dashboard |
| <br> `dashboard_tls_cert` | <br>  The public TLS certificate for the automation dashboard. This file must include the server certificate, intermediate CA certificates, and the root CA certificate. The server certificate must appear at the beginning of the file. |
| <br> `dashboard_tls_key` | <br>  The private TLS key for the automation dashboard. If not specified, the installation program generates a self-signed key. |
| <br> `postgresql_admin_username` | <br>  Admin username to access PostgreSQL database |
| <br> `postgresql_admin_password` | <br>  Admin password to access PostgreSQL database |
| <br> `registry_username` | <br>  Username used to pull container images from `registry.redhat.io`. This variable is required when using an online installation program. |
| <br> `registry_password` | <br>  Password used to pull container images from `registry.redhat.io`. This variable is required when using an online installation program. |
| <br> `dashboard_pg_containerized` | <br>  Configures the installation program to install and configure the database as a container on the same host as automation dashboard. `True` is also the only supported value. |
| <br> `dashboard_admin_password` | <br>  The password for the automation dashboard administrator user. The username is always `admin`. |
| <br> `dashboard_pg_host` | <br>  The hostname or IP address of the PostgreSQL database. This host must be accessible from the installation program node on port 5432. If a public IP address blocks this port, you must use the internal IP address. |
| <br> `dashboard_pg_username` | <br>  The database user for automation dashboard. |
| <br> `dashboard_pg_password` | <br>  The password for the `dashboard_pg_username`. |
| <br> `dashboard_pg_database` | <br>  The database schema name for automation dashboard. |
| <br> `bundle_install` | <br>  Indicates the required container images are already included in the installation bundle (tar file). |
| <br> `bundle_dir` | <br>  The directory where the installation bundle unpacks `+ /bundle` (for example: `/home/<username>/ansible-automation-dashboard-containerized-setup/bundle`). The default value is relative to the current directory (PWD) and should work without modification. |
| <br> `redis_mode` | <br>  Configures the Redis deployment mode. The required value is `standalone`. |
| <br> `registry_url_aap_automation_dashboard` | <br>  Configures a custom container registry URL specifically for the automation dashboard images. Use this if you must pull dashboard images from a different registry than your database or Redis images. |
| <br> `registry_ns_aap_automation_dashboard` | <br>  Configures a custom namespace specifically for the automation dashboard images. |
| <br> `registry_username_aap_automation_dashboard` | <br>  The username for the custom registry defined in `registry_url_aap_automation_dashboard`. Required if the dashboard registry differs from the primary registry. |
| <br> `registry_password_aap_automation_dashboard` | <br>  The password for the custom registry defined in `registry_url_aap_automation_dashboard`. Required if the dashboard registry differs from the primary registry. |
| <br> `registry_tls_verify_aap_automation_dashboard` | <br>  Enforces or disables TLS certificate verification when pulling automation dashboard images from a custom registry. Default: `True`. |
