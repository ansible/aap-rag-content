# 11. Projects
## 11.2. Updating projects from source control




Regularly updating your projects ensures that your environment has access to the latest versions of your playbooks, roles, and collections, reflecting any changes made in your Git, Subversion, or other integrated SCM repositories. This process is important for maintaining synchronization between your SCM and Ansible Automation Platform.

**Procedure**

1. From the navigation panel, selectAutomation Execution→Projects.
1. Click the sync![Sync](https://access.redhat.com/webassets/avalon/d/Red_Hat_Ansible_Automation_Platform-2.6-Using_automation_execution-en-US/images/8ca24410ba90c880998d13dfe67a52b0/sync.png)
icon next to the project that you want to update.

Note
Immediately after adding a project setup to use source control, a sync starts that fetches the project details from the configured source control.




- Click the project’s status under the **Status** column for further information about the update process. This brings you to the **Output** tab of the **Jobs** section.
