# 14. Event-Driven Ansible user scenarios
## 14.1. Simplified Event Routing using GitHub event streams
### 14.1.8. Check the Rule Audit for events on your new event stream

Checking the Rule Audit provides a detailed record of events associated with a specific event stream. This administrative view allows for the verification of successful event processing and the inspection of actions triggered by incoming payloads.

**Procedure**

1. From the Ansible Automation Platform navigation panel, select Automation Decisions → Rule Audit to see the event and actions.
2. Select the name of the event in the list that is associated with the rulebook activation you created. The Details tab is displayed.
3. Click the **Events** tab to view the event details.
4. Click the **Actions** tab to see what action took place.
5. If desired, click the **Name** of the action and view the **Details** tab to check the status.

**Results**

Your code is pushed and your applications are deployed.
